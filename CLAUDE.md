# CLAUDE.md

## デザイントークン

- 色の指定には必ずセマンティックカラートークンを使用する（`bg-primary`, `text-on-surface` 等）
- Tailwind のデフォルトカラー（`bg-blue-500`, `text-gray-600` 等）はビルドで無効化されているため使用不可
- 新しい色が必要な場合は `packages/ui/src/styles.css` の `@theme` にトークンを追加する
- 利用可能なトークン一覧は Storybook の「Foundations/Color Palette」で確認できる

## Storybook MCP

- UI コンポーネント開発時は Storybook MCP を活用してストーリーを作成すること
- Storybook dev サーバー（`cd packages/ui && bun run storybook`）の起動が前提条件

## コードレビュー

コミットや PR の前に、変更内容を自分でレビューする。lint・型チェック・テスト・ビルドは CI / lefthook がカバーしているので、ここでは「機械が拾えない問題」に集中する。

### コミット前（軽め）

コミットしてから「しまった」となる問題を防ぐ。深追いはしない。

- そのアプローチで合っているか？もっとシンプルな方法はないか
- 秘密情報のハードコード、デバッグコードの消し忘れ
- 見ればわかるバグ

### PR 作成前（批判的に）

マージ後に他の開発者に影響する問題を防ぐ。コミットレビューより踏み込んで見る。

- アプローチの妥当性：根本的な方向性は正しいか。既存の仕組みを活かせていないか
- ロジック：バグ、エッジケース、意図しない副作用
- セキュリティ：インジェクション、秘密情報、認証・認可
- 設計：不要な複雑さ、既存パターンとの不整合
- CLAUDE.md 準拠：このファイルのルールへの違反

### PR の種類

変更の性質から判断して適切な形式で PR を作成する。迷ったらユーザーに確認する。

- **auto-merge**: 小さな修正、リファクタ、設定変更など、レビュー不要で CI が通ればマージしてよいもの
- **通常**: 機能追加や設計変更など、他の開発者のレビューを待つべきもの
- **ドラフト**: WIP や意見を募りたいもの
